# HCG 제안서 PPT 디자인 가이드라인 v1.0

> **분석 기반:** 기아, KB라이프생명, NH투자증권, 크래프톤, 파라다이스, 하이트진로음료,
> 한화손해보험, 현대자동차, 리파인, 롯데알미늄 등 10개 제안서 / 약 530개 파일
> 피드백 텍스트 4개 + fb PPTX 8개 + 내부회의록 20+개 심층 분석

---

## 1. 핵심 원칙 (Core Principles)

| # | 원칙 | 근거 (실제 피드백 발췌) |
|---|------|----------------------|
| **P1** | **인사이트가 없으면 슬라이드가 아니다** | "너무 뻔한 얘긴데… 좀 업그레이드 해보세요 (형식 + 내용)" (한화fb) / "형식/내용을 컨설턴시 하게 재구성" |
| **P2** | **고객사 문맥 없는 방법론은 공허하다** | "우리 인사이트를 여기에 녹이는게 전략일수도" (기아fb) / "어떤 자동차/보상 프로젝트 경험 있는지 서술형으로 넣기" |
| **P3** | **슬라이드는 주장이다, 보고서가 아니다** | "메세지: 있는 현황은 시사점만 보였으면 좋겠음" / "헤드는 메세지 정리되면 현황을 정리해주기" |

## 2. 논리 구조 (Story Architecture)

### 2-1. 제안서 표준 목차 흐름

```
[Cover]
  └── [목차] 대챕터 3~4개만 (세부목차 X)
       ├── I.  제안 배경/현황 이해
       │    └── 산업/고객사 환경 → 이슈 가설 → HCG 관점 정리
       ├── II. 수행 방안
       │    └── Overall Approach → 단계별 방법론 → 핵심 차별성(Why HCG)
       ├── III.추진 계획
       │    └── 수행일정 → 투입조직 → 제안가격
       └── Appendix. 회사 소개 / 레퍼런스
```

### 2-2. 슬라이드 단위 논리 구조 (1 Slide = 1 Message)

```
┌─────────────────────────────────────────────────────┐
│  [헤드 메시지]  결론을 담은 완결 문장 (1~2줄)           │
├─────────────────────────────────────────────────────┤
│  본문 (Body)                                          │
│  • 3분법/4분법 도해 또는 프로세스 흐름도                 │
│  • 팩트/데이터 → 해석 → 시사점 흐름                    │
├─────────────────────────────────────────────────────┤
│  [선택] 각주: 데이터 출처 / 시점 명시                   │
└─────────────────────────────────────────────────────┘
```

### 2-3. 배경 슬라이드 전개 패턴

**틀린 흐름 (Don't):** `문제 나열 → 솔루션 제시`

**올바른 흐름 (Do):** `변화 상황 묘사 → 현상(징후/파열음) → 가설적 이슈 → HCG 관점 핵심 원인 → 대응 방향`

## 3. 헤드 메시지 작성 원칙

| 원칙 | Do | Don't |
|------|-----|-------|
| **결론 선행** | "HR ROI 저하는 보상-성과 연계 부재의 구조적 결과임" | "한화손해보험 현황 분석" |
| **중립성 유지** | "전략 연계형 인력운영 전환이 중장기 경쟁력의 핵심임" | "문제가 심각하므로 개선이 시급함" |
| **전체 통합** | 하단 2~3개 논거를 아우르는 하나의 명제 | 세부항목 중 하나만 반영한 좁은 헤드 |
| **오해 방지** | 헤드가 특정 요소만 지칭하는 듯 보이지 않도록 앞 내용과 연결 | "보상수준 = BM 2개만 활용"으로 오해 유발 |

## 4. 디자인 패턴 (Visual Design Standards)

### 4-1. 색상 체계

| 용도 | 색상 | 원칙 |
|------|------|------|
| **강조 / HCG 핵심 메시지** | 진한 파랑 (Deep Blue) | 슬라이드 내 1~2개 요소에만 |
| **보조 / 일반 요소** | 옅은 파랑 (Light Blue) | 구분 필요하되 강조 불필요한 요소 |
| **배경 / 기본값** | 회색 (Gray) | 기본이지만 덜 중요한 것에만. 중요 항목에 회색 금지 |
| **비교 강조** | 진한색(HCG) vs 회색(타사) | HCG 차별점 시각적으로 명확히 |

### 4-2. 레이아웃 원칙

| 원칙 | Do | Don't |
|------|-----|-------|
| **그룹핑 방식** | 선/화살표/+기호로 관계 표현 | 모든 것을 박스로 감싸기 |
| **비교 구조** | "기존 vs HCG" 좌우/상하 비교 | 동등 강조로 차별점 희석 |
| **정보 밀도** | 1 슬라이드 = 핵심 포인트 1~2개 | 모든 내용 한 장에 욱여넣기 |
| **슬라이드 구분** | `[참고]` 태그로 본문 vs 백업 구분 | 참고자료와 본문 혼용 |

### 4-3. 프로세스/도해 시각화 위계

```
Level 1 (최우선): Domain Knowledge → Analyze → Implication 3단계 흐름
Level 2:          거시환경 변화 → 산업 이슈 → 고객사 현황 깔때기 구조
Level 3:          수행단계(Phase) 기반 프로세스 흐름도
Level 4:          비교 매트릭스 (중요도 × 만족도 등)
```

### 4-4. 텍스트 처리

| 항목 | 기준 |
|------|------|
| **슬라이드 내 불릿 수** | 한 영역당 3~4개 이하 |
| **폰트/자간** | 전체 드래프트 완성 후 일괄 손보기 |
| **강조 텍스트** | 핵심 워딩만 볼드 or 색상 강조, 전체 강조 금지 |
| **각주** | 데이터 출처 + 기준 시점 반드시 명시 |

## 5. 콘텐츠 섹션별 작성 원칙

### 5-1. 제안 배경

| 항목 | 기준 |
|------|------|
| **첫 슬라이드** | 산업 현황 → 고객사 특유 현황 순서. "매출/이익/사건" 팩트 기반으로 시작 |
| **이슈 슬라이드** | 이슈의 발생 구조 (원인→현상→영향) 명확히 |
| **implication** | 별도 강조 없이 헤드 메시지에 자연스럽게 녹여서 |
| **백업 활용** | 세부 데이터/논거는 `[참고]` 슬라이드로 분리 |

### 5-2. Why HCG / 차별성 슬라이드

| 항목 | 기준 |
|------|------|
| **비교 구조** | "일반 접근(기본 요구수준) vs HCG(가치 창출)" 명확한 대비 |
| **기본 요소 처리** | 경쟁사/대안이 하는 것은 회색 처리 |
| **HCG 강점 3대 축** | ① 수행경험(레퍼런스 구체화) / ② Data Analytics / ③ 파트너십 |

### 5-3. 수행 방안 (방법론)

| 항목 | 기준 |
|------|------|
| **단계 수** | 4단계 표준 (Modeling → 실시 → 분석 → 개선방안) |
| **산출물** | 각 단계 주요 산출물 명시, 과도한 나열 삭제 |
| **레퍼런스 연결** | 방법론 단계마다 과거 유사 프로젝트 경험 1~2줄 삽입 |

### 5-4. 레퍼런스 / 수행사례 슬라이드

| 항목 | 기준 |
|------|------|
| **선정 기준** | 고객사 산업 or 유사 과업 기준으로 선정. 관련 없는 사례 삽입 금지 |
| **기재 방식** | 고객사명 + 과업 한 줄 + 주요 수행 내용 요약 (서술형 선호) |
| **그룹사 처리** | 그룹사 관련 프로젝트는 별도로 분류해 모으기 |
| **연도 처리** | 오래된/부적합 프로젝트 연도는 과감히 제거 |

## 6. Do & Don't 핵심 원칙 총람

### 6-1. 내용/논리 Do & Don't

| 구분 | Do ✅ | Don't ❌ |
|------|--------|----------|
| 헤드 메시지 | 결론/인사이트를 담은 완결 문장 | 슬라이드 제목 수준의 주제어 |
| 배경 전개 | 변화 → 현상 → 가설 이슈 흐름 | 문제 직접 나열 후 솔루션 제시 |
| 범위 설정 | 프로젝트 범위 내 구체적 내용 | "비전/전략/가치" 등 범위 초과 일반론 |
| 인사이트 표현 | HCG 고유 관점이 담긴 언어 | "~가 필요합니다" 류의 뻔한 결론 |
| 데이터 활용 | 출처/시점 명시한 구체 수치 | 출처 없는 임의 통계 |
| 산출물 기재 | 핵심 3~5개만 명시 | 전체 산출물 목록식 나열 |
| 레퍼런스 | 고객사 유관 프로젝트 서술형 | 관련성 낮은 프로젝트 로고 나열 |

### 6-2. 디자인 Do & Don't

| 구분 | Do ✅ | Don't ❌ |
|------|--------|----------|
| 색상 강조 | 진파랑=강조, 회색=기본, 1~2개 요소에 집중 | 중요 항목에 회색 처리 |
| 그룹핑 | 선/화살표/기호로 관계 표현 | 모든 개념을 박스로 묶기 |
| 정보 밀도 | 1 슬라이드 = 1 핵심 포인트 | 가능한 모든 정보를 한 장에 압축 |
| 백업 처리 | `[참고]` 태그로 본문과 분리 | 참고자료와 핵심 슬라이드 혼용 |
| 폰트/자간 | 전체 완성 후 일괄 통일 | 파트별 작업자가 각자 폰트 설정 |
| 비교 도해 | HCG=진색 / 경쟁대안=회색으로 시각 우세 | 동등 강조로 차별점 희석 |
| 목차 | 대챕터 3~4개 큰 목차 구성 | 세부 항목까지 목차에 모두 나열 |

## 7. 진화 과정 분석 (Draft → Final 변화 패턴)

| 단계 | 변화 방향 |
|------|----------|
| **Draft v0** | 방법론 중심, 내용 과다, 일반론 수준 |
| **피드백 후 v0.2~v0.5** | 고객사 맥락 구체화, 슬라이드 수 감소, 헤드 메시지 강화 |
| **피드백 후 v1** | 레퍼런스 연결, 일정/팀 최적화, 불필요 산출물 삭제 |
| **_final** | 텍스트 자간/폰트 통일, 디자인 세부 보정 |

## 8. 실무 적용 체크리스트

**논리 구조**
- [ ] 배경 → 이슈 → 방법론 → 계획 흐름이 자연스럽게 연결되는가?
- [ ] 각 슬라이드 헤드가 해당 장의 핵심 결론/인사이트를 담고 있는가?
- [ ] "뻔한 결론" 또는 "업계 공통 일반론"만 담긴 슬라이드는 없는가?
- [ ] 참고/백업 슬라이드와 본문 슬라이드가 명확히 구분되었는가?

**내용 품질**
- [ ] 고객사 특유 현황(산업/전략/데이터)이 방법론에 연결되어 있는가?
- [ ] HCG 레퍼런스는 고객사 과업과 유관한 사례로 선정되었는가?
- [ ] 과도한 산출물/일정이 "중압감"을 주지 않는가?
- [ ] 데이터/통계에 출처와 기준 시점이 명시되어 있는가?

**디자인**
- [ ] 강조 색상이 진파랑으로 1~2개 핵심 요소에만 집중되었는가?
- [ ] 중요한 항목이 회색 처리되지 않았는가?
- [ ] 폰트/자간이 전체 슬라이드에서 일관되게 통일되었는가?
- [ ] "HCG vs 경쟁대안" 비교 구조에서 HCG가 시각적으로 우세한가?

## 9. 섹션별 표준 슬라이드 매수

| 섹션 | 권장 슬라이드 수 | 비고 |
|------|---------------|------|
| 커버 + 목차 | 2장 | 목차는 대챕터만 |
| 제안 배경 (환경+이슈) | 3~5장 | 고객사 현황 1~2장 + 이슈 2~3장 |
| Why HCG | 1~2장 | 차별성 + 레퍼런스 |
| 수행 방안 (방법론) | 5~8장 | Overall + 단계별 상세 |
| 추진 계획 (일정+팀+가격) | 3장 | 각 1장 |
| Appendix (회사소개) | 3~4장 | |
| **합계** | **17~24장** | |
