# HCG 제안서 PPT 디자인 가이드라인 v2.0

> **분석 기반:** 기아, KB라이프생명, NH투자증권, 크래프톤, 파라다이스, 하이트진로음료,
> 한화손해보험, 현대자동차, 리파인, 롯데알미늄 등 10개 제안서 / 약 530개 파일
> fb PPTX 8개 + 피드백 텍스트 4개 + 내부회의록 20+개 + **롯데알미늄 _final vs Auto Draft 직접 비교**

## 0. 슬라이드 파일 스펙 (절대 준수)

| 항목 | 실제 표준값 | 틀린 설정 |
|------|-----------|---------|
| **슬라이드 크기** | **10.83" × 7.50"** (27.51 cm × 19.05 cm) | ~~13.33" (US 와이드스크린)~~ |
| **슬라이드 레이아웃 수** | 4개 (표지/목차/본문/End of document) | — |
| **본문 텍스트 폰트 크기** | **11pt** (세부 10pt) | ~~14pt 이상 이단 구조~~ |
| **본문 텍스트 스타일** | **Regular, #000000 (검정)** | ~~볼드, 파랑(#003887) 불릿~~ |
| **헤드 메시지 폰트** | **13~14pt, Regular 또는 Bold, Deep Blue** | — |
| **섹션 서브타이틀 크기** | **12~13pt** | — |

> **v1.0→v2.0 핵심 수정:** Auto Draft에서 본문 불릿이 전부 `11pt bold #003887`로 렌더링됨 → 실제 final은 `11pt black` (일반체). 헤드만 색상/볼드.

## 1. 핵심 원칙 (Core Principles)

| # | 원칙 | 근거 (실제 피드백 발췌) |
|---|------|----------------------|
| **P1** | **인사이트가 없으면 슬라이드가 아니다** | "너무 뻔한 얘긴데… 좀 업그레이드 해보세요 (형식 + 내용)" (한화fb) |
| **P2** | **고객사 문맥 없는 방법론은 공허하다** | "우리 인사이트를 여기에 녹이는게 전략일수도" (기아fb) |
| **P3** | **슬라이드는 주장이다, 보고서가 아니다** | "메세지: 있는 현황은 시사점만 보였으면 좋겠음" |
| **P4** | **AI는 빠른 표준 생성이 핵심이다** | "텍스트가 아니라 표준을 빠르게 만든다. 기존 가진 모델이 있고 거기에 AI-powered이고 고객 가이드를 넣어서 최대한 빠르게 만든다. 그게 핵심임." (롯데알미늄 fb) |
| **P5** | **Pay는 병렬이다, 순서가 없다** | "4P를 그냥 보여주면 됨. Pay level부터 볼 필요 없음. 꼭지별로 병렬적으로 보는 것임." (롯데알미늄 fb) |

## 2. 논리 구조 (Story Architecture)

### 2-1. 제안서 표준 목차 흐름 (25장 기준, 롯데알미늄 _final 실측)

```
Slide 1   : [Cover] 표지 레이아웃
Slide 2   : [목차] 대챕터 3~4개 (목차 레이아웃)
Slide 3   : Project Overview — 추진내용+프로세스+기간+인원+금액 1장 요약
Slide 4   : Client's Needs & Pain Point
Slide 5   : HCG's Approach (기존 접근 VS HCG)
Slide 6   : [참고] HCG 방법론 세부 (高맥락 컨설팅 등)
Slide 7   : 유사 프로젝트 수행 사례  ← ★ 방법론 앞에 배치
Slide 8   : 직무체계 개선    Overview  (3단계: Analysis / Classification / Evaluation)
Slide 9   : 직무체계 개선    Process
Slide 10  : 직무체계 개선    [세부 주제] 표준화 등
Slide 11  : [참고] 직무 관련 예시 / 분석 결과
Slide 12  : 평가제도 개선    Overview  (3단계: Criteria / Process / Application)
Slide 13  : 평가제도 개선    직군별 차별화 평가모델
Slide 14  : [참고] AI 기반 평가 시스템 예시 (talenx)
Slide 15  : 평가제도 개선    평가운영체계
Slide 16  : 보상제도 개선    Overview  (3단계: Pay Level / Structure / Simulation)
Slide 17  : 보상제도 개선    보상 지향점/정책선 설정
Slide 18  : 보상제도 개선    보상제도 설계
Slide 19  : 보상제도 개선    보상 Simulation
Slide 20  : [섹션 구분] 목차 레이아웃 (Appendix 진입 구분선)
Slide 21  : Appendix — HCG Overview
Slide 22  : Appendix — 사업 영역
Slide 23  : Appendix — 컨설팅 영역
Slide 24  : Appendix — 주요 고객사
Slide 25  : End of document
```

> **v1.0 오류 수정:**
> - v1.0은 "레퍼런스 = Appendix 뒤" → 실제는 **방법론 전 Slide 7에 배치**
> - v1.0은 투입조직/제안가격 각 1장 → 실제는 **Project Overview 1장에 통합**
> - v1.0은 Appendix = 3장 → 실제는 **4장 (Overview + 사업영역 + 컨설팅영역 + 고객사)**

### 2-2. 섹션 타이틀 포맷 규칙

```
[단일 텍스트박스]
"직무체계 개선    Overview"
  └── 앞: 섹션명 (Regular, 13pt, Black)
  └── 뒤: 서브타이틀 (Regular, 13pt, Black) — 탭/다중 스페이스로 간격
```

각 방법론 섹션은 반드시 `Overview → 세부 슬라이드 → [참고]` 순서로 전개.

### 2-3. [참고] 슬라이드 규칙

| 항목 | 기준 |
|------|------|
| **헤드 메시지** | **없음** — 타이틀만 (예: "[참고] AI 기반 업무 Check-in 시스템 활용 예시") |
| **내용** | 시각 예시 중심 (스크린샷, 도표, 샘플 산출물) |
| **위치** | 관련 본문 슬라이드 바로 다음 |
| **목적** | 설명하되 핵심 장표에 묻히지 않도록 분리 |

> **v1.0 오류:** [참고] 슬라이드에도 헤드 메시지를 작성했음 → 실제 final은 타이틀만 있음

### 2-4. Project Overview 1장 통합 원칙

Project Overview(Slide 3) 1장에 다음 4개 항목 전부 포함:
- **추진 내용**: 프로젝트 목적 1줄 요약
- **추진 프로세스**: Phase 1→2→3 흐름
- **추진 방안(잠정적)**: 각 Phase 핵심 액티비티
- **추진 기간 / 투입 인력 / 추진 금액**: 우측 하단 박스

## 3. 헤드 메시지 작성 원칙

| 원칙 | Do | Don't |
|------|-----|-------|
| **결론 선행** | "HR ROI 저하는 보상-성과 연계 부재의 구조적 결과임" | "한화손해보험 현황 분석" |
| **중립성 유지** | "전략 연계형 인력운영 전환이 중장기 경쟁력의 핵심임" | "문제가 심각하므로 개선이 시급함" |

| **[참고]는 예외** | [참고] 슬라이드는 타이틀만 작성 | [참고]에도 인사이트 헤드 작성 |

## 4. 디자인 패턴 (Visual Design Standards)

### 4-1. 색상 체계

| 용도 | 색상 | 원칙 |
|------|------|------|
| **헤드 메시지 / 섹션 강조** | Deep Blue (#003887 또는 유사) | 헤드 텍스트, 슬라이드 번호, 구분선 |
| **서브 강조** | Light Blue (#5B9BD5) | 보상 BM 시장선(Market Leading Line 등) |
| **본문 텍스트** | **#000000 (검정, Regular)** | 불릿 내용 전체 — 볼드/파랑 금지 |
| **기본/배경** | Gray (#808080) | 섹션 라벨, 시각화 힌트 |
| **비교 강조** | HCG=진파랑 / 타사=Gray | HCG 차별점 시각 우세 유지 |

### 4-2. 방법론 Overview 슬라이드 패턴 (3-Step 구조)

```
[섹션명    Overview]
[서브 헤드: 방법론 요약 1~2줄]

Step 1. [라벨]          Step 2. [라벨]          Step 3. [라벨]
  ↓ 핵심 항목              ↓ 핵심 항목              ↓ 핵심 항목
  세부 설명 10pt           세부 설명 10pt           세부 설명 10pt
```

3단계 구조 필수: 직무체계(Analysis/Classification/Evaluation), 평가(Criteria/Process/Application), 보상(Pay Level/Structure+Mix+Contributor/Simulation)

### 4-3. 보상 4P — 병렬 처리 원칙

```
보상 4P는 순서 없이 병렬 제시
┌──────────────┬──────────────┬──────────────┬──────────────┐
│  Pay Level   │Pay Structure │   Pay Mix    │Pay Contributor│
│  (외부경쟁력)│  (구조 설계) │  (비중 설계) │  (결정요인)  │
└──────────────┴──────────────┴──────────────┴──────────────┘
```

> fb 피드백: "Pay level부터 볼 필요는 없음. 꼭지별로 병렬적으로 보는 것임."

### 4-4. 평가 모델 분리 원칙

- **KPI 평가** (목표 달성 기반) → 전 직군 적용
- **업무 Check-in / 상시성과관리** → **사무직에 한함** (생산직 적용 불가)

> fb 피드백: "체크인은 사무직에 한함"

### 4-5. 텍스트 처리

| 항목 | 기준 |
|------|------|
| **본문 불릿 스타일** | Regular, 11pt, #000000 — 볼드/색상 강조 절대 금지 |
| **핵심 워딩 강조 시** | 볼드만 사용, 텍스트 전체에 색상 적용 금지 |
| **슬라이드 내 불릿 수** | 한 영역당 3~4개 이하 |
| **각주** | 데이터 출처 + 기준 시점 반드시 명시 |

## 5. 콘텐츠 섹션별 작성 원칙

### 5-1. 제안 배경

| 항목 | 기준 |
|------|------|
| **첫 슬라이드** | Pain Point → 기존 컨설팅 방식 한계 → HCG 해소 방향 흐름 |
| **Pain Point 그룹** | "파편화된 데이터" 군 + "유기적 연결 부족" 군으로 2분류 |
| **implication** | 헤드 메시지에 자연스럽게 녹여서 (별도 강조 불필요) |

### 5-2. HCG's Approach 슬라이드

| 항목 | 기준 |
|------|------|
| **핵심 메시지** | "AI + 전문가 Fine-Tuning = 빠른 표준 생성 → 고객과 Customization" |
| **AI 강조 포인트** | 속도 단축(60%)이 아니라 **표준을 빠르게 만든다**가 핵심 |
| **도해 구조** | 기존 방식(회색) vs HCG 방식(진파랑) 좌우 대비 필수 |
| **AI 도구** | JobSkillEngine (Claude API 기반, 직무분류체계·직무기술서·스킬디렉토리 자동생성) 언급 가능 |

### 5-3. 수행사례 / 레퍼런스 슬라이드

| 항목 | 기준 |
|------|------|
| **위치** | **방법론(수행방안) 섹션 직전** (Slide 7 위치) |
| **선정 기준** | 고객사 산업·유사 과업 기준 |
| **기재 방식** | 고객사명 + 과업 한 줄 + 주요 수행 내용 (서술형 선호) |
| **나열 방식** | 컴팩트 리스트형 (로고 그리드 X → 텍스트 목록) |

### 5-4. 직무체계 섹션

| 항목 | 기준 |
|------|------|
| **핵심 메시지** | 지주사 직무 분류 철학 계승 + AI 기반 글로벌 BM + 생산 현장 특수성 반영 |
| **슬라이드 구성** | Overview(3단계) → Process(직무분류/기술서/평가 흐름) → 표준화(공장별 현황→개선) → [참고] |
| **생산직 특화** | 공장별 R&R 불일치 → Global Standard 통합 메시지 명시 |

### 5-5. 평가제도 섹션

| 항목 | 기준 |
|------|------|
| **직군별 차별화** | R&D·영업·생산·경영지원 각각 다른 평가모델 명시 |
| **생산직 평가** | 절대평가 고려, 조직 성과 연계 성과급 |
| **AI 도구** | talenx 연계, 개인 목표/직무 입력 → KPI 자동 추천 |
| **체크인** | **사무직에만 적용** — 생산직에 check-in 적용 언급 금지 |

### 5-6. 보상제도 섹션

| 항목 | 기준 |
|------|------|
| **4P 배치** | Pay Level / Pay Structure / Pay Mix / Pay Contributor 병렬 제시 |
| **슬라이드 구성** | Overview(3단계) → 지향점/정책선(Market BM) → 제도 설계(직무/직급 차별화) → Simulation |
| **BM** | Market Leading(75%ile) / Matching(50%ile) / Following(25%ile) 3선 |
| **Simulation** | 전사 재무 Cost Simulation + 개인별 보상전환 Simulation 2트랙 |

## 6. Do & Don't 핵심 원칙 총람

### 6-1. 내용/논리 Do & Don't

| 구분 | Do ✅ | Don't ❌ |
|------|--------|----------|
| 헤드 메시지 | 결론/인사이트 담은 완결 문장 | 슬라이드 제목 수준의 주제어 |
| [참고] 슬라이드 | 타이틀만 작성, 비주얼 예시 중심 | [참고]에도 헤드 메시지 작성 |
| 레퍼런스 위치 | 방법론 섹션 직전 (Slide 7) | Appendix 뒤 배치 |
| Project Overview | 1장에 기간/인원/금액 통합 | 투입조직/가격 슬라이드 별도 분리 |
| 보상 4P | 병렬 제시 | Pay Level → Structure 순차 강제 |
| 체크인 | 사무직 한정 명시 | 전 직군 적용으로 서술 |
| AI 강조 포인트 | "표준을 빠르게 만들고 고객과 Customize" | "분석 시간 X% 단축" 수치만 강조 |
| 직무체계 연결 | 직무 분석 → KPI 도출 → 보상 연계 명시 | "직무중심 = 직무급 도입" 오해 유발 |

### 6-2. 디자인 Do & Don't

| 구분 | Do ✅ | Don't ❌ |
|------|--------|----------|
| 본문 텍스트 | 11pt, Regular, #000000 | 11pt bold #003887 (파랑 볼드) |
| 색상 강조 | 헤드 메시지·슬라이드번호에 Deep Blue | 본문 불릿 전체 색상 강조 |
| 섹션 타이틀 | "섹션명    서브타이틀" 단일 텍스트박스 | 별도 색상박스로 분리 |
| 슬라이드 크기 | 10.83" × 7.50" | 13.33" (US 와이드스크린) |
| 목차 | 대챕터 3~4개 큰 목차 | 세부 항목까지 목차에 나열 |
| Appendix 구분 | 목차 레이아웃으로 섹션 구분 슬라이드 삽입 | 본문 슬라이드로 그냥 시작 |
